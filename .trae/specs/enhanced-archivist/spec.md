# Enhanced Archivist Spec

## Why
The current `Archivist` lacks support for complex contract structures (Volumes, Signature Pages, Attachments) and does not provide sufficient semantic context (full paths, summaries) for the Supervisor agent.

## What Changes
- **Hierarchy Detection**: Introduce a depth-weighted regex system to handle multi-level structures (Volume -> Chapter -> Section -> Clause).
- **Special Page Handling**: Recognize "Signature Page" as a top-level node.
- **Chunking Logic**: Ensure headers are included at the beginning of chunks for better context.
- **Metadata Enhancement**:
    - Convert `path` to a `/`-separated string.
    - Generate semantic summaries for each chunk using an LLM (Worker model).

## Impact
- **Affected Code**: `src/core/archivist.py`.
- **Affected Specs**: Previous specs relying on `path` as a list might need updates (e.g., `retriever.py` or `lookup.py` if they expect a list, but user requested string path).
    - Note: `Retriever` was previously updated to handle list paths by joining them. Now we will do it in `Archivist`.
    - `generate_document_structure` in `Archivist` expects list paths to build the tree. We should update it to handle string paths or keep the internal representation as list but output metadata as string.
    - Decision: Keep internal tracking as list for tree generation, but store string in `metadata['path']`.

## ADDED Requirements

### Requirement: Depth-Weighted Hierarchy Detection
The system SHALL support the following hierarchy levels with regex matching:
1.  **Depth 1**: Volume/Book/Part (`第X卷/册/篇`), Special Pages (`此页为合同签字页`).
2.  **Depth 2**: Chapter (`第X章`), Attachment (`附件X`).
3.  **Depth 3**: Section (`X.X`).
4.  **Depth 4**: Clause (`X.X.X`).

Logic:
- **WHEN** a header is detected with Depth $N$.
- **THEN** truncate the current path to keep elements with Depth < $N$.
- **AND** append the new header.

### Requirement: Semantic Chunking
- **Header Alignment**: When a new header is detected, it SHALL be the first line of the new chunk's content.
- **Summary Generation**: Each chunk SHALL include a `summary` field in metadata, generated by the Worker model (limit 50 words).

### Requirement: Metadata Formatting
- **Path String**: The `path` in metadata SHALL be a string joined by `/` (e.g., "第二卷/第四章/4.2").

## MODIFIED Requirements
- **generate_document_structure**: Must adapt to the new hierarchy logic (if implementation details change).
